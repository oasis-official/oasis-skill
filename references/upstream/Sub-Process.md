# Sub-Process

**프로세스 컨텍스트를 분리**하거나 **프로세스를 재사용**하기 위해서 서브 프로세스를 사용할 수 있다. 서브 프로세스는 인라인 서브프로세스, 오프라인 서브프로세스로 구분할 수 있고 인라인 서브프로세스는 재사용 할 수 없다.

# 인라인 서브프로세스

프로세스 흐름 내 연결된 서브프로세스이다. 

### 서브 프로세스 생성

도구 상자에서 *Create extended SubProcess* 를 클릭하여 만들고 다른 요소와 *연결 도구(Flow)*을 이용해서 연결한다.

![Untitled](Sub-Process/Untitled.png)

![Untitled](Sub-Process/Untitled%201.png)

서브 프로세스는 **프로세스 컨텍스트**가 분리되고 상위 컨텍스트(서비스 컨텍스트, 애플리케이션 컨텍스트)는 공유한다. 즉, 클라이언트에서 전달한 값이나 애플리케이션 단 정보는 공유하지만 메인 프로세스 내에서 만들어진 데이터는 서브 프로세스와 공유하지 않는다.

## 서브 프로세스에 참조 공유

메인 프로세스의 프로세스 컨텍스트 전체를 공유하지 않지만 선택적으로 서브 프로세스에 참조를 공유할 수 있다.

![Untitled](Sub-Process/Untitled%202.png)

프로세스를 클릭하여 *Extensions* 탭에서 *input* 속성을 추가한다. 속성값은 컨텍스트에서 가져올 키인데 *PropertyEL*을 사용할 수 있다.

# 오프라인 서브프로세스

프로세스 흐름 밖에 선언되어 있는 서브프로세스이다. 서브 프로세스는 접혀진 서브 프로세스 태스크를 통해서 호출 할 수 있다.

### 서브 프로세스 호출 태스크 생성

서브 프로세스 호출을 위해 태스크를 *SubProcess* 로 변경한다. 변경된 태스크는 하단에 + 마크가 나타나 서브 프로세스 임을 알린다.

![Untitled](Sub-Process/Untitled%203.png)

![Untitled](Sub-Process/Untitled%204.png)

### 서브 프로세스 생성

실제 실행할 서브 프로세스를 생성한다. 도도구 상자에서 *Create extended SubProcess* 를 클릭하여 만들고 다른 요소와 연결을 만들지 않는다.

![](Sub-Process/Untitled%205.png)

## 서브 프로세스 실행과 데이터 전달

![Untitled](Sub-Process/Untitled%206.png)

접혀진 서브프로세스 *Extensions* 탭에서 *processId* 속성으로 실행할 서브프로세스Id를 지정하고,  *input* 속성을 추가한다. 속성값은 컨텍스트에서 가져올 키인데 *PropertyEL*을 사용할 수 있다.

<aside>
💡 피 호출 Process 에도 `input` 프로퍼티를 설정할 수 있다. 이 경우 부모 컨텍스트에서 지정한 데이터를 능동적으로 가져올 수 있는데 우선순위는 실행하는 태스크의 input 보다 낮다.

input/output 탭에서 데이터를 직접 입력한 경우 호출한 서브 프로세스 컨택스트의 입력값과 피 호출 프로세스의 값은 병합되어 실행된다. 이 경우도 피 호출 프로세스에 지정한 값의 우선순위가 낮아 같은 키를 가진 값이 있다면 호출한 서브 프로세스 태스크의 입력값이 더 우선한다.

피 호출 프로세스는 `output`을 지원하지 않는다. 호출 프로세스에 직접 명시해야한다.

</aside>
