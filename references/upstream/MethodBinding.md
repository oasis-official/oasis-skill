# MethodBinding

*JavaServiceTask*는 *Java Class* 속성에 지정한 클래스의 객체를 생성하고 *method* 프로퍼티에 지정한 메소드를 호출하는 태스크이다. 이 페이지는 어떤 전략으로 생성자를 선택하여 객체를 생성하고, 메소드를 호출하는지에 대한 설명을 한다.

# 객체 생성

지정한 메소드를 호출하기 위해서는 지정한 클래스의 객체를 생성해야 한다. 객체를 생성하기 위해서는 객체 생성에 사용할 생성자를 선택해야하는데 생성자가 한 개만 존재할 경우 문제가 없지만 여러개 인경우 어떤 생성자를 사용해야 할지 결정을 해야한다.

## 결정 과정

1. 후보 생성자 결정
2. 파라미터 길이가 긴 생성자에서 적은 순으로 정렬
3. 생성자 파라미터 매칭 전략에 따른 생성자 결정

### 후보 생성자 결정

모든 public 생성자를 후보 생성자로 본다.

### 정렬

파라미터가 긴 생성자를 먼저오도록 정렬한다.

```java
class MyClass {
    public MyClass(String name, String address, int age) {
    }

    public MyClass(String name, int age) {
    }

    public MyClass(String name) {
    }
}
```

### 파라미터 매칭

1. 파라미터 이름으로 컨텍스트에 데이터가 있는지 확인하고
2. 없으면 타입으로 호환가능한 데이터가 있는지 확인한다. 

<aside>
💡 타입은 사용자가 만든 클래스에 한하여 매칭한다. 즉, `String`, `Map` 등 자바에서 제공하는 타입과 매칭은 하지 않는다.

</aside>

1. 모든 파라미터가 컨텍스트와 매칭이 되면 해당 생성자를 선택한다.

# 메소드 호출

지정한 메소드가 클래스에 한개만 존재하면 별 문제가 없지만 메소드 오버로딩으로 이름은 같고 파라미터가 다른 메소드가 존재하면 어떤 메소드를 호출해야 할지 결정해야한다. 상세 결정 과정은 생성자를 선택하는 기준과 같다.
